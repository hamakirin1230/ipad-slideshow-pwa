## 1. プロジェクトの目的

この案件は、iPadで安定して閲覧・操作できるスライドショーPWAを作るプロジェクトである。

最優先は、学校現場・イベント現場の本番中に止まらないこと。

そのため、初期版では派手な演出や高度なAI機能より、次を優先する。

* iPadホーム画面PWAで安定して起動・操作できること
* オフライン再生へ進める構成を崩さないこと
* 同期状態をユーザーに分かりやすく見せること
* 誤操作を防ぐこと
* Google Drive権限を最小限にすること
* GitHub Pages上で壊れない静的PWA構成にすること

## 2. 技術前提

* Next.js


第3ゴールでは、Google OAuth Token model の検証を完了した。

* `docs/decisions.md` に第1〜第3の設計判断・完了確認・今後守る制約を追記

第3の重要方針:

* Google scopeは `https://www.googleapis.com/auth/drive.file` のみ
* `openid` / `email` / `profile` は使わない
* `drive` / `drive.readonly` / `drive.metadata` / `drive.appdata` は使わない
* Google公式スクリプト `https://accounts.google.com/gsi/client` は `/auth-test` のみで読み込む
* `access_token` は `useRef` にだけ保持
* `access_token` を画面表示しない
* `access_token` をconsole出力しない
* `access_token` をlocalStorage / IndexedDB / OPFS / Cookieに保存しない
* リロード後・PWA開き直し後は未接続に戻る
* 接続状態リセットはGoogle側のrevokeではなく、画面内の一時token削除のみ
* Client Secretは作らない、使わない
* Client IDの実値はコード、docs、チャット、workflowに直書きしない
* GitHub Pages公開版ではRepository Variablesからbuild時に `NEXT_PUBLIC_GOOGLE_CLIENT_ID` を渡す

Google Identity Services Token model のリクエスト方針:

```ts
scope: "https://www.googleapis.com/auth/drive.file"
prompt: "select_account"
include_granted_scopes: false
```

GitHub Pagesのoriginとして、以下は入れない。


```text

```

`.env.local` はGit管理対象外。commitしない。

GitHub Pages公開版では、GitHub Repository Variablesに以下を設定済み。

```text
NEXT_PUBLIC_GOOGLE_CLIENT_ID
```

GitHub Actionsのbuild時に、Repository VariablesをNext.jsへ渡す設定済み。

```yaml
env:
  NEXT_PUBLIC_GOOGLE_CLIENT_ID: ${{ vars.NEXT_PUBLIC_GOOGLE_CLIENT_ID }}
```

## 12. 第4以降で守る制約

第4以降では、次を守る。

* Google scopeは原則 `drive.file` のまま進める
* `profile` / `email` / `openid` を安易に追加しない
* `drive` や `drive.readonly` を安易に追加しない
* `access_token` を永続保存しない
* tokenの実値を画面、console、docs、ログに出さない
* Client Secretを作らない、使わない
* Drive API操作は、第4で目的を絞って段階的に追加する
* iPadホーム画面PWAで失敗したものは完了扱いにしない
* GitHub Pagesの `basePath` 前提を崩さない
* 公開版で必要な `NEXT_PUBLIC_` 値は、GitHub Repository Variablesからbuild時に渡す
* `.env.local` は各PCで作るが、GitHubには上げない
* Windowsローカルでは `http://localhost:3000/ipad-slideshow-pwa/...` を使う
* `http://127.0.0.1:3000`、`http://0.0.0.0:3000`、`http://192.168.x.x:3000` はOAuth確認に使わない
